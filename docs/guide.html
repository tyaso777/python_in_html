Python in HTML - User Guide

1. Overview

Python in HTML runs Python directly in the browser using Pyodide. You can load input files, edit Python code in cell form, run one cell or all cells, and inspect per-operation results including text output, images, and downloadable files.

Main features:

  • No local Python installation required
  • Input files are available under /inputs/
  • Generated files are available under /outputs/
  • Cell-based execution with # %%
  • Config save/load/restore support
  • Per-operation result history on the right side
  • Image preview and download links for generated outputs

2. Basic Workflow

  1. Choose input files in the Input files panel
  2. Edit the code in the Python code editor
  3. Run the current cell or all cells
  4. Check Operation Results for stdout, stderr, images, and downloads

3. Input Files

Input files are selected with the Choose Files button. Supported file types in the current UI are .csv, .json, .txt, .xlsx, and .xls.

Selected files are stored in the browser-side virtual file system under:

/inputs/your_file_name

Example:

import pandas as pd

df = pd.read_csv("/inputs/sample_user.csv")
File names must match exactly. For example, sample_user.csv and sample-user.csv are different names.

Excel Compatibility Mode

Excel files are supported through the app compatibility adapter. After you choose an .xlsx or .xls file, use its internal path under /inputs/ with pd.read_excel(...).

This app runs fully in the browser with Pyodide. In this environment, the usual desktop-style pandas Excel backend is not used directly, so Excel workbooks are parsed on the JavaScript side first and then exposed to Python through a pd.read_excel(...)-compatible adapter. That is why common usage works, while some advanced cases can still differ from native desktop pandas. 

import pandas as pd

df = pd.read_excel("/inputs/祝日.xlsx", sheet_name=0)
all_sheets = pd.read_excel("/inputs/祝日.xlsx", sheet_name=None)

The adapter is designed to follow common pandas.read_excel usage for local browser work.

  • Supported patterns include sheet_name, header, names, index_col, usecols, skiprows, nrows, dtype, parse_dates, na_values, true_values, and false_values
  • sheet_name=None returns a dict keyed by sheet name, matching normal pandas behavior
  • The Input files panel shows the internal path used by Python
This is compatibility mode, not the native pandas Excel engine. Most common workflows should work, but some advanced read_excel options may still differ from desktop pandas.

4. Output Files

Files written to /outputs/ are treated as generated results.

  • Image files such as .png appear in Images
  • All generated files appear in Downloads
plt.savefig("/outputs/plot.png")
df.to_csv("/outputs/result.csv", index=False)

5. Cell-Based Execution

Use # %% to split code into cells. Run Current Cell executes the cell that contains the cursor. Run All Cells executes cells from top to bottom and records each cell as a separate result card. 

# %%
print("cell 1")

# %%
print("cell 2")

6. Keyboard Shortcuts

Shortcut Action
Shift + Enter Run the current cell
Ctrl + Enter Run the current cell and move to the next visible cell
Tab Indent selected code
Shift + Tab Outdent selected code

If Ctrl + Enter is used on the last visible cell, a new # %% cell is created at the end and the cursor moves into it.

7. Configuration

The Configuration panel lets you control Python packages and save or load your current setup.

  • Save Config: export the current code and selected packages as JSON
  • Load Config: load a previously saved JSON config
  • Restore Default: restore the built-in default code and package set
  • Show Config / Hide Config: expand or collapse the full configuration UI

Package selection is limited to an allowlist of Pyodide built-in packages.

8. Package Filters

In the package picker, packages can be filtered by three axes at the same time:

  • Level: Recommended, Standard, Advanced, All
  • Domain: Analysis, ML, Plot, Text, DB, Image, Geo, Science, Utility, All
  • License Review: Generally OK, Review Needed, All

The search box works together with all filters.

Each package row also shows a short normalized license label such as MIT, BSD, or Apache-2.0, together with its License Review tag.

How License Review is determined

License Review is a practical UI classification based on package metadata. It is meant as a conservative screening aid for internal use, not as legal advice.

  • Generally OK: assigned when the package only matches commonly accepted permissive-style labels in this app's normalization step, such as MIT, BSD, Apache-2.0, PSF, MPL-2.0, AFL, or BSL-1.0
  • Review Needed: assigned when the normalized result includes a value that should be checked manually, such as Unknown, GPL, LGPL, AGPL-3.0-or-Commercial, Freely Distributable, Public Domain, or any-OSI

If a package has multiple detected licenses, it is marked as Review Needed when any one of them falls into the manual-review group.

The initial filter state is Generally OK, so the package list starts in the more conservative view by default.

9. Operation Results

Every action is recorded on the right side as an operation card.

Examples:

  • Choose Files
  • Delete File
  • Run Current Cell n
  • Reset Session
  • Restore Default

For cell execution, each card may contain:

  • Stdout / stderr
  • Images
  • Downloads

10. Sample Use Case

Sample CSV files are stored in the repository under samples/. After you select them in the app, read them from /inputs/.

Use these sample files:

  • sample_user.csv
  • sample_purchase.csv
import pandas as pd
import numpy as np
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt

users = pd.read_csv("/inputs/sample_user.csv")
purchases = pd.read_csv("/inputs/sample_purchase.csv")

df = pd.merge(purchases, users, on="user_id", how="left")

summary = (
    df.groupby("user_name")["amount"]
    .sum()
    .reset_index()
    .sort_values("amount", ascending=False)
)

print(summary)

plt.figure(figsize=(8, 4))
plt.bar(summary["user_name"], summary["amount"])
plt.title("Total Purchase Amount by User")
plt.xlabel("User")
plt.ylabel("Amount")
plt.xticks(rotation=30)
plt.tight_layout()
plt.savefig("/outputs/user_total.png", dpi=140)

summary.to_csv("/outputs/user_summary.csv", index=False)

11. Helpful Commands

If you are unsure which input files are currently available, run:

import os
print(os.listdir("/inputs"))

If you want to inspect generated files, run:

import os
print(os.listdir("/outputs"))

12. Notes and Limitations

All processing happens in the browser. Files are handled in the in-browser virtual file system used by Pyodide.
  • Large files may be slow because processing uses browser memory
  • Input files and runtime state disappear when the page is reloaded
  • Only files under /inputs/ are treated as input files
  • Only files under /outputs/ appear as generated results
  • Package choices are limited to the built-in allowlist defined by this app
  • Excel loading uses the app compatibility adapter rather than the native pandas Excel backend

13. Troubleshooting

FileNotFoundError

The file path in Python does not match the selected input file name. 

import os
print(os.listdir("/inputs"))

No image appears

Make sure the image is written under /outputs/. 

plt.savefig("/outputs/plot.png")

No download link appears

Make sure the file is written under /outputs/.

Load Config does not restore a package

Only packages included in the app's allowlist are restored from config files.

Excel results do not match desktop pandas exactly

Check the Excel Compatibility Mode section. This app supports common read_excel usage, but some advanced cases can still differ from native pandas.
